# Databridge

A multi-source data viewer and reporting tool. Simulates pulling data from multiple source systems (CRM, order management, warehouse), normalizing it through an Airflow ETL pipeline into a central Postgres database, and presenting it through a REST API and Svelte dashboard.

**Stack:** FastAPI · Svelte · Postgres · Airflow · Celery · Redis · LocalStack · Docker

---

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)

That's it. No Python, Node, or anything else required locally.

---

## Getting Started

**1. Clone the repo**
```bash
git clone git@github.com:gretschduojet1/databridge.git
cd databridge
```

**2. Start the stack**
```bash
./start.sh
```

This script:
- Creates `.env` from `.env.example` if one doesn't exist (generates a real `SECRET_KEY` automatically)
- Starts all Docker services and waits for them to be healthy
- Seeds the users table so you can log in immediately
- Seeds the raw landing-zone tables (200 CRM customers, 15 WMS products, 500 OMS transactions, 150 warehouse stock rows) ready for Airflow to ingest

First run pulls images and installs dependencies — takes a few minutes. Subsequent starts are fast.

> **Note:** Auto-creating `.env` is a convenience for this demo project. In a real environment you would provision secrets through your deployment platform (e.g. AWS Parameter Store) rather than deriving them from a file in the repo.

**3. Run the Airflow ETL DAGs**

Open Airflow at **http://localhost:8080** (credentials: `admin` / `admin`) and trigger the DAGs in order:

| DAG | What it does |
|---|---|
| `ingest_customers` | CRM → `customers.customers` |
| `ingest_products` | WMS → `inventory.products` |
| `ingest_orders` | OMS → `sales.orders` (requires customers + products) |
| `ingest_warehouse_stock` | Warehouse dump → `stores.locations` + `stores.stock` |
| `compute_stock_projections` | Computes stockout forecasts from sales velocity |

Each DAG uses an `ingested_at` watermark — rows with `ingested_at IS NULL` are unprocessed. Re-running a DAG is safe; it only picks up new rows.

The API docs at `/docs` are auto-generated by FastAPI — every endpoint is interactive and testable from the browser.

**Authenticating in Swagger:**
1. Expand `POST /auth/login` → click **Try it out** → select a credential from the Examples dropdown → **Execute**
2. Copy the `access_token` from the response
3. Click **Authorize** (lock icon, top right) → paste the token → **Authorize**

All subsequent requests in that session will include the token automatically.

**Using Postman:** Import `http://localhost:8000/openapi.json` directly (File → Import → Link) to generate a full collection from the OpenAPI spec.

## LocalStack (AWS Emulation)

LocalStack runs S3, SSM, and SES locally so the full AWS-integrated workflow works without a real AWS account.

**It is optional.** Without it, export jobs fall back to emailing the file as a direct attachment — all other features work normally.

To enable LocalStack, sign up at [localstack.cloud](https://localstack.cloud) (free tier available — no credit card required) and add your token to `.env`:
```
LOCALSTACK_AUTH_TOKEN=your_token_here
```

The init script at `localstack/init/01_bootstrap.sh` runs automatically on startup and creates the S3 bucket and SSM parameters.

To use real AWS instead of LocalStack, set in `.env`:
```
AWS_ENDPOINT_URL=
S3_PUBLIC_URL=
```
And ensure the host has valid AWS credentials (IAM role, `~/.aws/credentials`, or environment variables).

## Database

The database uses multiple schemas to simulate distinct source systems:

| Schema | Tables | Simulates |
|---|---|---|
| `raw` | `crm_customers`, `wms_inventory`, `oms_transactions`, `warehouse_stock` | Raw landing zone (unnormalized source exports) |
| `customers` | `customers` | Normalized CRM data |
| `sales` | `orders` | Normalized order data |
| `inventory` | `products`, `stock_projections` | Normalized product data + Airflow-computed forecasts |
| `stores` | `locations`, `stock` | Store inventory levels |
| `auth` | `users` | Identity / access |
| `workers` | `jobs` | Background job tracking |

**ETL flow:** `seed_raw.py` populates the `raw` schema. Airflow DAGs read from `raw`, transform and normalize, and write to the target schemas. The `ingested_at` column on every raw table is the watermark — `NULL` means unprocessed.

To connect directly:
```bash
docker compose exec db psql -U databridge -d databridge
```

## Security

- Passwords hashed with bcrypt
- JWT authentication on all non-public endpoints (30 minute expiry)
- Role-based access: `admin` and `viewer`
- Login endpoint rate limited to 5 requests/minute
- CORS locked to the frontend origin
- Security headers on all API responses (CSP, X-Frame-Options, Referrer-Policy)

## Roadmap

**Phase 1 — Docker + Database + Seed Data** ✅
- Three-service Docker stack (db, backend, frontend)
- Postgres schema with three simulated source systems
- FastAPI with repository pattern, Pydantic DTOs, Protocol interfaces
- Seed script generating realistic sample data
- Test infrastructure with pytest

**Phase 2 — Auth + API Completion** ✅
- JWT authentication with role-based access
- Rate limiting on login
- Svelte login page with protected app shell
- Sorting, filtering, and pagination on all list endpoints
- Aggregation endpoints (sales totals, inventory summary, revenue over time)
- Alembic database migrations

**Phase 3 — Svelte UI** ✅
- Split-panel login page with gradient branding
- Dark sidebar navigation with icons and active state highlighting
- Dashboard with color-coded stat cards and Chart.js visualizations
- Bar chart (revenue by region) and line chart (monthly revenue trend)
- Table views for customers, products, and orders
- Sorting, filtering (region, category, date range), and server-side pagination on all tables
- Excel export on every data view
- Badge components for regions, categories, and roles

**Phase 4 — Background Jobs** ✅
- Event dispatch system: API routes fire events, workers pick them up asynchronously
- Job queue with Celery + Redis
- Tasks: summary report generation, simulated customer sync from upstream CRM
- Job status tracking (pending → running → success/failed) with Postgres persistence
- Sweeper task: automatically re-enqueues stuck pending/running jobs every 60 seconds
- Dataset export to Excel: dispatched as a background job, result emailed via MailHog
- Writer abstraction (`WriterProtocol`) with Excel and text implementations, swappable in one place
- Service container (`core/container.py`): all interface→implementation bindings centralized
- Jobs page in the UI: dispatch panel, live status polling, result/error display, active/failed counts in sidebar
- Flower monitoring dashboard at http://localhost:5555

**Phase 5 — Airflow ETL + Stores** ✅
- Raw landing-zone schema (`raw.*`) simulating unnormalized exports from CRM, WMS, and OMS
- Four Airflow DAGs: ingest customers, products, orders, and warehouse stock
- `ingested_at` watermark pattern — each raw row processed exactly once, safe to re-run
- Stores page: per-store inventory with search, region filter, low stock indicators, and detail panel
- Dashboard inventory projections: Airflow computes stockout forecasts nightly from 365-day sales velocity
- Projection table shows avg units/day, days until stockout, projected date, and velocity trend (accelerating / steady / slowing)

**Phase 6 — AWS Integration (LocalStack)** ✅
- LocalStack running S3, SSM, and SES — full local AWS emulation, no account required for basic use
- Export jobs upload files to S3 (`databridge-exports` bucket) instead of attaching to emails
- Pre-signed download URLs emailed to the requesting user (24-hour expiry)
- App fetches `SECRET_KEY`, `DATABASE_URL`, `REDIS_URL` from SSM Parameter Store at startup — falls back to env vars if SSM is unavailable
- `AWS_ENDPOINT_URL` and `S3_PUBLIC_URL` are env-var driven so the same codebase works against LocalStack or real AWS without code changes
- Export jobs degrade gracefully — uploads to S3 and emails a pre-signed link when available, falls back to emailing the file as an attachment if S3 is unreachable
- Swagger UI auth switched to HTTP Bearer — copy a token from `/auth/login` and paste it into Authorize
- Login endpoint shows clickable example credentials in Swagger (`openapi_examples`)
- Login page demo credential rows are clickable — selecting one fills the form automatically
- `start.sh` seeds raw tables automatically so Airflow DAGs are ready to run on first boot
